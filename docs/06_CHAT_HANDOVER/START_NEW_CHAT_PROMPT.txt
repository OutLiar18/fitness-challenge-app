# ABOUT THE PROJECT

Champions Legacy Challenge is **not** simply a fitness tracker.

It is a gamified personal development platform focused on helping people become better than they were yesterday.

The platform encourages:

• Discipline

• Consistency

• Accountability

• Physical Fitness

• Reading

• Skill Development

• Healthy Habits

• Community

• Personal Growth

Every feature should reinforce these principles.

# DEVELOPMENT PHILOSOPHY

Build systems.

Do not build one-off solutions.

Whenever possible:

- Avoid hardcoded values.
- Prefer configuration over duplication.
- Create reusable components.
- Separate business logic from UI.
- Keep services modular.
- Think about future admin functionality.
- Consider analytics before designing data structures.

Every system should be designed so it can grow without major rewrites.

# ADMIN SYSTEM

The project will eventually include a comprehensive administration system.

Administrators should be able to:

- Review submissions
- Approve evidence
- Reject evidence
- Award bonus points
- Correct mistakes
- Manage users
- Adjust challenge settings
- Search participants
- View analytics
- Manage competitions

Design current systems so they naturally support future administration features.

---

# SCORING

The points system is one of the most important parts of Champions Legacy.

Whenever possible:

Store factual information.

Calculate scores from that data.

Avoid permanently storing calculated values unless there is a compelling performance reason.

Administrators should always be capable of overriding scores.
