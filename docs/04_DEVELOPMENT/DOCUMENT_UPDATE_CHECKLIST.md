# Champions Legacy

# Document Update Checklist

Version: 2.0

---

# Purpose

Documentation is considered part of the software.

Whenever implementation changes, the relevant documentation should be reviewed and updated.

This checklist ensures that documentation remains accurate, consistent and trustworthy throughout the lifetime of Champions Legacy.

---

# Philosophy

Outdated documentation is often more harmful than missing documentation.

Whenever code changes, ask:

"What documentation is affected?"

Updating documentation should be part of completing a feature—not something done later.

---

# When to Use This Checklist

Use this checklist whenever:

- A feature is added.
- A feature is removed.
- Architecture changes.
- Rules change.
- User experience changes.
- Configuration changes.
- A release is prepared.

---

# Final Review

Before considering work complete:

□ Documentation matches implementation.

□ No outdated information remains.

□ Cross references verified.

□ Version information updated.

□ New documents added if required.

---

# Guiding Principles

Before closing any task, ask:

Have I documented this change?

Will another developer understand it?

Will Future Me understand it six months from now?

Is the documentation still trustworthy?

If the answer to any question is "No", continue updating documentation.
