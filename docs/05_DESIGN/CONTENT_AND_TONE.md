# Champions Legacy

# Content and Tone

Version: 2.0

---

# Purpose

This document defines how Champions Legacy communicates with its users.

Every word displayed within the application should reinforce the project's identity, values and mission.

Consistent language builds trust, improves usability and creates a more personal experience.

---

# Philosophy

Champions Legacy is a guide, not a drill sergeant.

The application should encourage users without shaming them.

It should celebrate progress without exaggeration.

Its tone should always reflect respect for the player's effort.

---

# Brand Personality

Champions Legacy communicates as though it is:

- Encouraging
- Professional
- Calm
- Honest
- Motivating
- Supportive
- Confident

It should never sound:

- Condescending
- Aggressive
- Sarcastic
- Childish
- Overly corporate
- Manipulative

---

# Writing Principles

Every piece of text should be:

## Clear

Use simple language.

Avoid unnecessary jargon.

---

## Concise

Communicate only what the user needs.

Avoid long paragraphs where short sentences are sufficient.

---

## Positive

Focus on progress rather than failure.

Instead of:

"You failed today's goal."

Prefer:

"You still have time to complete today's goal."

---

## Human

Write naturally.

Avoid robotic phrasing.

The application should feel like a knowledgeable coach rather than a machine.

---

# Button Labels

Buttons should clearly describe the action they perform.

Good examples:

- Save Entry
- View Progress
- Continue
- Complete Challenge
- Edit Profile

Avoid:

- OK
- Submit
- Execute
- Confirm

---

# Success Messages

Success should acknowledge effort.

Examples:

- Great work!
- Entry saved successfully.
- Another step towards your goals.
- Consistency beats perfection.
- Nice job—keep the momentum going.

Success messages should remain sincere and avoid becoming repetitive.

---

# Error Messages

Error messages should explain:

- What happened.
- Why it happened.
- How to fix it.

Good:

"Please enter a valid distance."

Poor:

"Error 402."

---

# Empty States

When no information exists, guide the user.

Examples:

"No journal entries yet."

"Start today's first challenge."

"Your progress begins with one small step."

Empty screens should encourage action rather than appear unfinished.

---

# Notifications

Notifications should:

- Be relevant.
- Be timely.
- Avoid unnecessary interruption.

The application should avoid excessive reminders.

---

# Achievement Messages

Achievements should celebrate effort.

Examples:

"You've completed your first week!"

"Consistency is becoming a habit."

"Excellent progress."

Achievements should feel meaningful rather than constant.

---

# Motivation

Champions Legacy rewards effort.

Avoid comparing players negatively.

Focus on:

- Personal growth.
- Improvement.
- Consistency.
- Healthy competition.

---

# Competitive Language

Competition should inspire.

Avoid language that humiliates lower-ranked players.

Examples:

Good:

"Only 12 points behind first place."

Poor:

"You're losing badly."

---

# Validation Messages

Validation should be helpful.

Examples:

"Distance must be greater than zero."

"Please select an exercise."

"The amount cannot be empty."

Avoid technical terminology.

---

# Confirmation Dialogues

Confirm only important actions.

Examples:

Delete Entry

Leave Without Saving

Reset Progress

Avoid confirmation for routine actions.

---

# Loading Messages

Loading should reassure users.

Examples:

"Loading your progress..."

"Updating leaderboard..."

"Saving your entry..."

---

# Tone Across the Application

The tone should remain consistent regardless of screen.

Whether viewing:

- Journal
- Statistics
- Leaderboards
- Challenges
- Profile

The application should always sound like Champions Legacy.

---

# Decision Checklist

Before writing any interface text ask:

Is it clear?

Is it encouraging?

Is it concise?

Would a new user understand it?

Does it sound like Champions Legacy?

If the answer is "No", rewrite it.
