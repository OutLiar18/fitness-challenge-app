# Champions Legacy

# Design Language

Version: 2.0

---

# Purpose

The Design Language defines the visual identity of Champions Legacy.

It establishes a shared visual vocabulary so that every screen, component and interaction feels like part of the same product.

Rather than prescribing exact implementation details, this document defines the principles that guide visual design.

---

# Philosophy

The visual design should communicate confidence, clarity and progress.

Every design element should have a purpose.

Visual consistency strengthens the Champions Legacy identity and creates a more intuitive user experience.

---

# Brand Personality

Champions Legacy should feel:

- Modern
- Professional
- Motivating
- Calm
- Trustworthy
- Premium
- Clean

The interface should avoid appearing childish, cluttered or overly decorative.

---

# Colour Philosophy

Colours communicate meaning before text is read.

They should be used consistently throughout the application.

General guidelines:

- Primary colours represent actions.
- Success colours represent achievement.
- Warning colours indicate caution.
- Error colours communicate problems.
- Neutral colours support readability.

Colour should reinforce meaning rather than provide decoration.

---

# Typography Philosophy

Typography establishes hierarchy.

Text should always prioritise readability over style.

Headings should clearly separate sections.

Body text should remain comfortable to read across all supported devices.

---

# Spacing

Consistent spacing improves visual rhythm.

Avoid arbitrary margins and padding.

Whitespace is an intentional design element that improves readability and reduces clutter.

---

# Shapes

Rounded corners should create a modern, approachable appearance.

Shape consistency should be maintained across:

- Buttons
- Cards
- Inputs
- Dialogs
- Navigation elements

---

# Elevation

Elevation should communicate hierarchy.

Use shadows sparingly to distinguish interactive or layered elements.

Avoid excessive visual depth.

---

# Icons

Icons should:

- Support understanding.
- Complement text.
- Remain visually consistent.
- Avoid replacing labels where clarity may be reduced.

Icons should improve usability rather than decoration.

---

# Motion

Animations should:

- Explain changes.
- Reinforce interactions.
- Feel smooth.
- Remain subtle.

Motion should never delay users unnecessarily.

---

# Imagery

Images should support the application's purpose.

Prefer authentic imagery over generic stock photography.

Illustrations should maintain the same visual style throughout the application.

---

# Visual Hierarchy

Visual emphasis should generally follow:

1. Primary action
2. Current objective
3. Progress
4. Supporting content
5. Decorative elements

Users should immediately understand where to focus their attention.

---

# Consistency

Every screen should appear as though it belongs to the same application.

Consistency applies to:

- Colours
- Typography
- Icons
- Layout
- Buttons
- Forms
- Feedback
- Navigation

---

# Scalability

The design language should support future growth.

New screens and features should naturally integrate with the existing visual system without introducing conflicting styles.

---

# Decision Checklist

Before introducing a new visual element, ask:

Does it improve clarity?

Is it consistent with the existing interface?

Does it reinforce the Champions Legacy identity?

Does it serve a clear purpose?

Would users immediately understand it?

If the answer is "No", reconsider the design.
