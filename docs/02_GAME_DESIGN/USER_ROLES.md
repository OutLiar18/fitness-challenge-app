# Champions Legacy

# User Roles

Version: 2.0

---

# Purpose

The User Roles System defines the different roles that players and organisers may hold within Champions Legacy.

Roles determine a person's responsibilities within leagues and communities while helping maintain fairness, integrity and enjoyable competition.

Roles should grant responsibility rather than status.

---

# Design Philosophy

Every player should begin on equal footing.

Additional roles exist only to support the community, organise competitions and protect the integrity of the platform.

Roles should never create unfair gameplay advantages.

---

# Core Objectives

The User Roles System has five primary objectives.

## 1. Define Responsibilities

Each role should have clearly understood responsibilities.

---

## 2. Maintain Fairness

Roles should help preserve fair competition rather than influence results.

---

## 3. Support Communities

Roles should encourage organisation, communication and accountability.

---

## 4. Protect Integrity

Certain roles should be able to investigate disputes, clarify rules and resolve issues fairly.

---

## 5. Enable Growth

The role system should support future expansion without requiring fundamental redesign.

---

# Core Role Structure

Champions Legacy currently recognises the following primary roles:

- Player
- League Administrator
- Platform Administrator

Future roles may be introduced as the platform grows.

---

# Player

Every registered user is a Player.

Players form the foundation of Champions Legacy.

Players are responsible for:

- Participating honestly.
- Recording activities accurately.
- Following challenge rules.
- Treating others respectfully.
- Reporting issues where appropriate.

Players should have equal opportunity to succeed.

---

# League Administrator

League Administrators organise and manage individual leagues.

Responsibilities include:

- Creating leagues.
- Inviting players.
- Managing league settings.
- Clarifying league-specific rules.
- Resolving disputes.
- Reviewing reported issues.
- Maintaining a positive environment.

League Administrators should remain impartial.

Administrative authority should never be used to gain a competitive advantage.

---

# Platform Administrator

Platform Administrators oversee Champions Legacy as a whole.

Responsibilities may include:

- Managing platform-wide rules.
- Balancing game systems.
- Investigating abuse.
- Managing reports.
- Supporting league administrators.
- Maintaining platform integrity.

Platform Administrators should act consistently and transparently.

---

# Temporary Responsibilities

Certain responsibilities may be assigned temporarily.

Examples include:

- Event Organiser
- Tournament Host
- Community Moderator
- Beta Tester

Temporary responsibilities should not permanently alter a player's role.

---

# Future Roles

Future versions may introduce additional roles such as:

- Team Captain
- Mentor
- Coach
- Ambassador
- Content Creator
- Verified Organisation
- Official Event Host

Each new role should have a clearly defined purpose.

---

# Permissions Philosophy

Permissions should always follow the Principle of Least Privilege.

Users should only receive the permissions necessary to fulfil their responsibilities.

Additional permissions should never exist without clear justification.

---

# Leadership

Leadership roles should exist to serve the community.

Leaders should:

- Encourage participation.
- Support new players.
- Promote fairness.
- Resolve conflict respectfully.
- Lead by example.

Leadership is demonstrated through actions rather than authority.

---

# Accountability

All roles are accountable for their actions.

Administrative decisions should be:

- Fair
- Consistent
- Transparent
- Documented where appropriate

No role should be exempt from platform rules.

---

# Conflict Resolution

Disagreements should be resolved respectfully.

Where possible, issues should be handled at the lowest appropriate level.

Examples include:

- Player discussion.
- League Administrator review.
- Platform Administrator review.

Escalation should occur only when necessary.

---

# Fair Competition

Roles must never influence gameplay outcomes.

Examples of prohibited behaviour include:

- Awarding unfair advantages.
- Manipulating scores.
- Ignoring rule violations.
- Favouring specific players.

Trust is essential to maintaining healthy competition.

---

# Community Behaviour

Every role should promote a welcoming environment.

Expected behaviours include:

- Respect.
- Honesty.
- Encouragement.
- Sportsmanship.
- Constructive communication.

Negative behaviour weakens the community regardless of a person's role.

---

# Role Progression

Roles represent responsibility rather than achievement.

Becoming a League Administrator or Platform Administrator should be based on trust, reliability and commitment rather than points or player level.

Administrative roles are not rewards.

---

# Future Expansion

Future versions may support:

- Community moderation teams.
- Verified organisations.
- Educational institutions.
- Corporate wellness leagues.
- Charity organisers.
- Volunteer mentors.

These additions should remain consistent with the existing role philosophy.

---

# Relationship to Other Systems

User Roles support the operation of multiple systems.

These include:

- Challenge Rules
- League System
- Player Profile System
- Achievements & Rewards

User Roles define responsibilities.

They do not alter gameplay mechanics.

---

# Guiding Principles

Every future role should answer the following questions.

Does it improve the player experience?

Does it help the community?

Does it preserve fairness?

Does it avoid unnecessary complexity?

Does it align with the philosophy of Champions Legacy?

If the answer to any question is "No", the role should be reconsidered.

---

# Changes Since Version 1.0

Version 2.0 formalises the role structure for Champions Legacy.

The system now distinguishes between players, league administrators and platform administrators while providing a scalable framework for future community roles.

Roles are defined by responsibility rather than privilege, ensuring that leadership strengthens the platform without compromising fairness.
